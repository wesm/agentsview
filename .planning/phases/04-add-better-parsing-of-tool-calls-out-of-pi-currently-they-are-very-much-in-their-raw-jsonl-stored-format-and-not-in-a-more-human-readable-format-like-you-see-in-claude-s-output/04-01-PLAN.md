<objective>
Improve pi agent tool call rendering so pi tools display with the same visual polish as Claude Code tools, and suppress the empty user message bubbles caused by toolResult entries.

Purpose: Pi tool calls currently fall through to generic key:value fallback output because their raw JSONL names (str_replace, run_command, create_file, read_file) are not recognized by the existing tool rendering infrastructure. This causes a degraded experience for pi users.

Output: Four files modified (taxonomy.go, content-parser.ts, tool-params.ts, MessageList.svelte) plus updated tests.
</objective>
